# 环境配置清单

本清单描述 `ts-foundry-template` 的工具层、职责边界和版本来源。具体版本以 `pnpm-workspace.yaml` 的 catalog 和 `pnpm-lock.yaml` 为准；本文记录工具在母版中的工程职责。

## 运行底座

| 领域 | 默认值 | 职责 |
| :--- | :--- | :--- |
| Node runtime | Node 24 | 本地开发、CLI、Hono API、构建工具运行时 |
| Package manager | pnpm 10 | workspace、catalog、lockfile、安装策略 |
| Monorepo orchestration | Turborepo 2 | 任务图、增量缓存、跨包执行 |
| Module format | ESM | 应用、library、CLI 的默认模块格式 |

Node 24 是当前母版底座。母版不维护旧 Node 兼容层，不为 CommonJS 设置默认输出，也不为多个 runtime 同时设计安装路径。

## 语言与类型

| 领域 | 默认值 | 职责 |
| :--- | :--- | :--- |
| Language | TypeScript 6 | 类型边界、strict 配置、声明输出 |
| Shared tsconfig | `packages/tsconfig` | `base`、`react`、`node`、`lib`、`test` 分环境配置 |
| Boundary schema | Zod 4 | API、表单、shared contracts、运行时校验 |

TypeScript 负责静态边界，Zod 负责运行时边界。跨包协议不只导出 TypeScript 类型，也应提供可执行 schema，使 API、表单和测试都能验证真实输入输出。

## 质量工具

| 领域 | 默认值 | 职责 |
| :--- | :--- | :--- |
| Format / lint | Biome 2 | 格式化、lint、import organize |
| Unit / integration test | Vitest 4 | 单元测试、集成测试、coverage |
| Coverage provider | v8 | 覆盖率收集 |
| Browser E2E | Playwright | 浏览器关键路径 |
| Dependency audit | Knip | unused deps、unlisted deps、unresolved imports、unused files |
| Version consistency | syncpack | catalog 和 package 版本一致性 |
| Git hooks | Lefthook | pre-commit、pre-push 门禁 |
| Dependency updates | Renovate | 依赖升级分组 |
| Versioning | Changesets | package 版本和发布预留 |

Biome 是唯一默认格式化和 lint 入口。Vitest 是默认测试入口。Playwright 只覆盖浏览器关键路径。Knip 和 syncpack 共同保证依赖不漂移。

## 前端模板栈

| 领域 | 默认值 | 职责 |
| :--- | :--- | :--- |
| UI runtime | React 19 | 前端和 fullstack UI |
| Frontend build | Vite 8 | 开发服务和生产构建 |
| Routing | TanStack Router | 类型化路由、params、search 边界 |
| Server state | TanStack Query | 异步读取、缓存、失效 |
| Form | React Hook Form | 表单状态 |
| Form/schema binding | Zod resolver | 表单运行时校验 |
| Style | Tailwind CSS v4 | 样式 token、布局、组件样式 |
| Motion | Motion | 有语义的交互动画 |
| DOM test | Testing Library | React 组件和用户交互测试 |
| Browser mock | MSW | 网络边界 mock |

React 只属于前端和 fullstack 模板，不进入纯后端、CLI 或 library 默认依赖。Tailwind 和 Motion 同样只属于 UI 相关模板。

## 后端模板栈

| 领域 | 默认值 | 职责 |
| :--- | :--- | :--- |
| API framework | Hono | Web 标准 API 服务 |
| Node adapter | `@hono/node-server` | Node 24 HTTP server |
| Contract | Zod schemas | request/response 校验 |
| API test | Vitest + Hono request | handler 和错误路径验证 |

Hono API 模板覆盖 `/health`、列表读取、详情读取、创建请求和错误响应。后端模板不引入 React、Tailwind、Playwright 或浏览器测试。

## Library 与 CLI 栈

| 领域 | 默认值 | 职责 |
| :--- | :--- | :--- |
| Build | tsdown | ESM、类型声明、CLI bin 输出 |
| Dev runner | tsx | 脚本和 CLI 开发运行 |
| Package checks | publint、arethetypeswrong | 发布边界和类型包检查 |
| Test | Vitest | 函数、main 入口、错误码测试 |

library 模板输出可消费 ESM 包和类型声明。CLI 模板输出 ESM bin，并验证文本输出、JSON 输出和错误码。

## Mock 与测试数据

| 领域 | 默认值 | 职责 |
| :--- | :--- | :--- |
| Network mock | MSW 2 | 前端测试和 e2e 的 API 边界 |
| Fake data | faker 10 | seed 后的测试数据 |
| Shared utilities | `packages/test-utils` | faker seed、MSW helper、测试辅助函数 |

faker 必须 seed。测试中的随机数据不能影响可复现性。MSW worker 是前端模板运行文件，保留在 `public/mockServiceWorker.js`，但从 Biome 和 Knip 的源码检查中排除。

## 版本来源

新增共享依赖时，应先进入 `pnpm-workspace.yaml` catalog，再在各 package 中使用 `catalog:`。根部 devDependencies 也使用 catalog，保证母版只有一个版本控制面。

`pnpm-lock.yaml` 是安装结果的事实记录。CI 使用 frozen lockfile，不允许隐式修改 lockfile。

## 排除项

本清单不配置数据库、ORM、迁移工具、Docker、部署平台、SSR、Next.js、NestJS、生产发布 workflow、远程缓存或云服务。这些属于复制后实例项目的业务技术选型。
